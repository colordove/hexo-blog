Vue2开发体验

  • JavaScript
  • Vue

前言

使用vue-cli搭建项目，集合了vue-router，vuex等，UI库使用了饿了么的Mint-UI，Vue一如既往的小巧灵敏易上手，我这个从Angular1过来的几乎是直接拿来写代码，学习成本很低，比我切到Angular2好像容易的多，再吐槽下Angular2，感觉学习曲线又上了一个台阶。。。(p.s.也可能是语法不熟悉的缘故)。vue的单文件组织我很喜欢，只是目前编辑器支持依然不够好，之前在知乎向vs code一位开发者反馈，之后版本更新高亮问题解决了，但格式化和代码提示依然是痛。。。

我15年还是个naive的student时也差不多是这个月份，考驾照边等教练边用手机看vue1的语法，所以虽然之前没正式开发过Vue项目，但语法已经很熟悉了orz。。

开始

安装vuc-cli

sudo npm install -g vue-cli

初始化项目

vue init webpack [my-project]

安装依赖

cd [my-project] && npm install

开发

npm run dev

目录结构

src开发目录
  • assets —— 静态文件资源，包括图片、样式等
  • components —— vue组件
  • store —— vuex，包括actios，mutations等vuex的那套
  • utils —— 我的工具类函数，包括filter、rsa加密、封装的ajax请求等
  • views —— 页面视图
  • main.js —— 引导函数
  • App.vue —— 主视图
  • routers.js —— 路由

路由

vue-router首先要在App.vue里写上，有萌新这个都不写就追着问为什么他的路由不跳转。。
<router-view></router-view>

在项目中使用到子路由，另外可以利用meta传参做许多很方便的操作，比如隐藏掉tab等，还可以使用一些路由钩子函数做一些事，mode设为history时路由中的#将会被替换。我的路由大概是这样子的。

Vuex

在原有的Angular1项目中，随着项目逐渐增大，利用rootScope用来做controller间的通信已经变得很难维护，在Vue中直接引入Vuex，vue-devtools可以很方便的观察到数据流向，有一个小问题是我直接this.$store.state = [state]时vue-devtools没有检测到store变化，不知道是不是bug。

vuex提供了mapActions等辅助函数，可以使用ES6的…来展开在实例中调用。

  • store.js —— 用来声明需要维护的状态
  • actions.js —— 用来提交mutations
  • mutations.js —— 用来改变store中的state
  • mutation_types.js —— mutations.js中的一些常量
  • getters —— 用来获取state

Vue-resources

虽然尤大已经推荐axios为官方推荐的ajax库了，然而由于之前就封装好了，所以我还是继续使用了vue-resources，Vue.http.interceptors中方法可以在请求前完成一些其他操作，我将user_id和user_token加在了这里。

p.s.之前在请求v2ex的api时请求返回了一个json数组，vue-resources愉快的给我甩出JSON解析错误，懒的理这个就直接换axios了[逃]

Mint-ui

老实说我看到VUX有点尴尬，mint-ui其实还不错。希望组件能更丰富一些，不过这种，稍大些的公司都要自己定制一套吧。

Lodash

话说order by没有了唉，引入Lodash方便很多，尤其对数据有较多操作的时候，真心方便多了，推荐一个啊，吼。然后很多过滤器也没有了，要自己写。。

小结

Vue2小而美，但其实我更喜欢Angular那种标准的大而全的框架，毕竟想要正常工作总是要搭一套的，整体下来几乎没什么区别了。。。

但是，Vue容易上手，学习简单，对于团队来说会很赞，尤其是Vue有一种替代jQuery的感觉。。预计Vue在2016年的快速发展还会延续到2017年。前几天看到vue-cli已经可以创建weex项目了，也希望weex能尽快发展吧，现在大家都不敢碰weex。。。
